# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

**Do NOT report security vulnerabilities through public GitHub issues.**

Please use [GitHub Security Advisories](https://github.com/tschm/jquantstats/security/advisories) instead:

1. Go to the Security Advisories page linked above.
2. Click **"New draft security advisory"**.
3. Fill in the details and submit.

### What to Include

- **Description**: A clear description of the vulnerability
- **Impact**: The potential impact
- **Steps to Reproduce**: Detailed steps to reproduce the issue
- **Affected Versions**: Which versions are affected
- **Suggested Fix**: If you have one (optional)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution**: We aim to resolve critical issues within 30 days
- **Credit**: Reporters will be credited in the advisory unless anonymity is requested
